# <a name="whats-new-or-changed-in-dynamics-365-talent-february-14-2019"></a>Kas nauja ar pasikeitė sistemoje „Dynamics 365 Talent“ (2019 m. vasario 14 d.)

[!include [banner](includes/banner.md)]

Šioje temoje aprašomos naujos ir pakeistos „Talent“ funkcijos.

## <a name="changes-in-attract"></a>„Attract“ pakeitimai
Į šį leidimą įtraukta nedidelių pataisų.

## <a name="changes-in-onboarding"></a>Supažindinimo pakeitimai
Į šį leidimą įtraukta nedidelių pataisų.
 
## <a name="changes-in-core-hr"></a>„Core HR“ pakeitimai 
**8.1.2146 versija**

### <a name="employee-fixed-compensation-entity-doesnt-export-all-records"></a>Darbuotojo pastoviosios atlyginimo dalies objekto negalima eksportuoti į visus įrašus
Atlikus šį pakeitimą, objektas **Darbuotojo pastovioji atlyginimo dalis** galės eksportuoti visus įrašus. Objektą galima naudoti norint sukurti ir atnaujinti esamus darbuotojų pastoviųjų atlyginimo dalių įrašus. 

### <a name="employment-end-date-doesnt-honor-employee-preferred-time-zone-settings"></a>Įdarbinimo pabaigos data nustatoma neatsižvelgiant į darbuotojo pageidaujamos laiko juostos parametrus
Įdarbinimo pabaigos datos dabar nustatomos atsižvelgiant į vartotojo pageidaujamą laiko juostą, kai įmonėje kuriamas arba baigiamas įdarbinimo procesas.
 
### <a name="uk-addresses-display-in-analytics-as-eastern-switzerland-addresses"></a>JK adresai „Analytics“ rodomi kaip Rytų Šveicarijos adresai
Šiame leidime atliktas pakeitimas, kuriuo ištaisyti adresų nesutapimai **personalo valdymo** ataskaitoje Darbuotojų skaičiaus pagal vietą.
 
### <a name="termination-code-is-not-populated-on-the-worker-position-assignment-record-when-ending-the-position"></a>Atleidimo kodas neužpildomas darbuotojo pareigų priskyrimo įraše, kai pareigų ėjimas nutraukiamas
Kai nutraukiamas darbuotojų pareigų priskyrimas, nustatomas numatytasis kodas Atleidimo priežastis.

### <a name="new-entity-created-for-job-compensation-levels"></a>Sukuriamas naujas objektas, skirtas užduoties kompensacijų lygiams
Sukurtas naujas duomenų valdymo sistemos (DMF) objektas. Objektas suteikia galimybę kurti ir naujinti kiekvienos sistemoje nurodytos užduoties kompensacijų lygius, rinkos vertes ir apklausų informaciją.
