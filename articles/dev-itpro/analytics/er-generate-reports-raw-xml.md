# <a name="generate-reports-by-adding-content-as-raw-xml"></a><span data-ttu-id="97c84-103">Generuoti ataskaitas įtraukiant turinį kaip neapdorotą XML</span><span class="sxs-lookup"><span data-stu-id="97c84-103">Generate reports by adding content as raw XML</span></span>

[!include[banner](../includes/banner.md)]

<span data-ttu-id="97c84-104">Galite naudoti naują formato elementą **NEAPDOROTAS XML**, skirtą kurti elektroninių ataskaitų (ER) formatus, generuojančius siunčiamus dokumentus XML formatu.</span><span class="sxs-lookup"><span data-stu-id="97c84-104">You can use the new **RAW XML** format element to design Electronic reporting (ER) formats that generate outgoing documents in XML format.</span></span> <span data-ttu-id="97c84-105">Kai kuriais atvejais galite pageidauti įtraukti neapdorotus XML duomenis į šias ataskaitas dėl vienos ar daugiau iš šių priežasčių:</span><span class="sxs-lookup"><span data-stu-id="97c84-105">In some cases, you might prefer to add raw XML data to these reports for one or more of the following reasons:</span></span>

- <span data-ttu-id="97c84-106">Patogiau naudoti neapdorotą XML originaliam ataskaitos dizainui kurti ir vykdomai priežiūrai atlikti, nes XML struktūrą galima automatiškai sugeneruoti vykdant apdorojimo laiko išraišką.</span><span class="sxs-lookup"><span data-stu-id="97c84-106">It's more convenient to use raw XML for the original design and ongoing maintenance of a report, because the XML structure can be automatically generated by executing a runtime expression.</span></span> <span data-ttu-id="97c84-107">Todėl kuriant nereikia nustatyti kelių susiejimų keliems formato elementams.</span><span class="sxs-lookup"><span data-stu-id="97c84-107">Therefore, multiple bindings don't have to be determined for multiple format elements at design time.</span></span> <span data-ttu-id="97c84-108">Tai įmanoma, kai naudojate duomenų šaltinius, apimančius informaciją, kurią galima naudoti XML elementams daryti, kol generuojama ataskaita.</span><span class="sxs-lookup"><span data-stu-id="97c84-108">It is possible when the data sources that you're using contain information that can be used to make XML elements while the report is generated.</span></span>
- <span data-ttu-id="97c84-109">Jokių kitų metodų negalima naudoti pildant ataskaitą XML turiniu, kuris anksčiau buvo gautas ir saugotas sistemoje.</span><span class="sxs-lookup"><span data-stu-id="97c84-109">No other method can be used to fill the report with XML content that was previously received and stored in the system.</span></span> <span data-ttu-id="97c84-110">Pvz., sugeneruotame XML atsakyme gali būti turinio iš XML užklausos, kuri buvo išsiųsta anksčiau.</span><span class="sxs-lookup"><span data-stu-id="97c84-110">For example, the XML response that is generated might have to contain the content of an XML request that was sent earlier.</span></span>
- <span data-ttu-id="97c84-111">Jokio kito metodo negalima naudoti simboliams į sugeneruotą dokumentą įterpti pagal jų skaitinius kodus.</span><span class="sxs-lookup"><span data-stu-id="97c84-111">No other method can be used to insert characters into the generated document based on their numeric codes.</span></span> <span data-ttu-id="97c84-112">Kai kuriose kalbose ir tarp simbolių šio tipo kodų nėra.</span><span class="sxs-lookup"><span data-stu-id="97c84-112">For some languages and characters, codes of this type don't exist.</span></span> <span data-ttu-id="97c84-113">Pavyzdžiai apima graikų kalbos raidę ro (ρ) ir HTML objekto kodus, pvz., \&eacute; yra *e*, kuriame yra aiškus kirtis (é).</span><span class="sxs-lookup"><span data-stu-id="97c84-113">Examples include the Greek letter rho (ρ) and HTML entity codes such as \&eacute; for an *e* that has an acute accent (é).</span></span>

> [!NOTE]
> <span data-ttu-id="97c84-114">Turėkite omenyje, kad sistema nekontroliuoja, ar XML turinys, kuris įvedamas į sugeneruotą dokumentą naudojant formato elementą **NEAPDOROTAS XML** yra teisingas.</span><span class="sxs-lookup"><span data-stu-id="97c84-114">Be aware that the framework doesn't control whether the XML content that is placed to the generated document by using the **RAW XML** format element is correct.</span></span>
