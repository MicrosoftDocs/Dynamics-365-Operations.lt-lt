# <a name="set-up-and-use-pay-when-paid-vendor-payments"></a><span data-ttu-id="e165d-103">„Mokėti sumokėjus“ tiekėjo mokėjimų nustatymas ir naudojimas</span><span class="sxs-lookup"><span data-stu-id="e165d-103">Set up and use pay-when-paid vendor payments</span></span>

[!include [banner](../includes/banner.md)]

<span data-ttu-id="e165d-104">Kai patvirtinate tiekėją ir jam leidžiate dirbti kaip subrangovui, mokėjimą tiekėjui galite sulaikyti tol, kol jums už projektą sumokės jūsų klientas.</span><span class="sxs-lookup"><span data-stu-id="e165d-104">When you approve a vendor to work as a subcontractor, you might want to withhold payment to the vendor until your customer pays you for the project.</span></span> <span data-ttu-id="e165d-105">Pagal šį scenarijų, sąlygas „mokėti sumokėjus‟ (PWP) galite nustatyti tada, kai su tiekėju nustatote pirkimo užsakymą (PU).</span><span class="sxs-lookup"><span data-stu-id="e165d-105">To support this scenario, you can set up pay-when-paid (PWP) terms when you set up the purchase order (PO) with the vendor.</span></span>

<span data-ttu-id="e165d-106">Pavyzdžiui, kai klientas sumoka projekto SF sumą, galite išleisti dalį tiekėjo SF sumos arba visą sumą.</span><span class="sxs-lookup"><span data-stu-id="e165d-106">For example, when a customer pays an amount on a project invoice, you can release some or all of the vendor invoice amount.</span></span> <span data-ttu-id="e165d-107">Tiesiog nustatykite tokias PWP sąlygas, kuriomis nurodoma, kad tiekėjui bus sumokėta po to, kai iš kliento gausite susijusio mokėjimo procentą.</span><span class="sxs-lookup"><span data-stu-id="e165d-107">Just set up PWP terms that specify that the vendor will be paid after you receive a percentage of the related payment from the customer.</span></span> <span data-ttu-id="e165d-108">Jei iš kliento gaunate dalinį mokėjimą, galite rankiniu būdu apmokėti išleisti kai kurias susijusias tiekėjo SF.</span><span class="sxs-lookup"><span data-stu-id="e165d-108">If you receive partial payment from a customer, you can manually release some of the related vendor invoices for payment.</span></span>

<span data-ttu-id="e165d-109">Toliau pateiktas pavyzdys apibūdina procesą, kai naudojamos PWP sąlygos.</span><span class="sxs-lookup"><span data-stu-id="e165d-109">The following example outlines the process when PWP terms are used.</span></span>

## <a name="example"></a><span data-ttu-id="e165d-110">Pavyzdys</span><span class="sxs-lookup"><span data-stu-id="e165d-110">Example</span></span>

<span data-ttu-id="e165d-111">Jūsų organizacija sutinka pateikti klientui 100 kompiuterių, kuriuose įdiegta programinė įranga. Vieno kompiuterio kaina – 150 JAV dolerių (USD).</span><span class="sxs-lookup"><span data-stu-id="e165d-111">Your organization agrees to provide a customer with 100 computers that have software installed, for a price of 150.00 US dollars (USD) per computer.</span></span> <span data-ttu-id="e165d-112">Tada jūs pasamdote tiekėją, kuris jums pateiks kompiuterius su įdiegta programinė įranga.</span><span class="sxs-lookup"><span data-stu-id="e165d-112">You then hire a vendor to provide you with the computers that have software installed.</span></span> <span data-ttu-id="e165d-113">Pagal sutartį klientas turi patvirtinti kompiuterių kokybę prieš sumokėdamas jūsų organizacijai.</span><span class="sxs-lookup"><span data-stu-id="e165d-113">According to the agreement, the customer must approve the quality of the computers before it pays your organization.</span></span> <span data-ttu-id="e165d-114">Jūsų organizacijos strategija yra sustabdyti mokėjimą tiekėjams, kol jums sumokės klientas.</span><span class="sxs-lookup"><span data-stu-id="e165d-114">Your organization's policy is to withhold payment to vendors until the customer has paid you.</span></span> <span data-ttu-id="e165d-115">Todėl nustatote projektą taip, kad jo PWP procentas būtų 100 procentų.</span><span class="sxs-lookup"><span data-stu-id="e165d-115">Therefore, you set up the project so that it has a PWP percentage of 100 percent.</span></span>

<span data-ttu-id="e165d-116">Tiekėjas atsiunčia jums 100 kompiuterių, kuriuose įdiegta programinė įranga, kartu su sąskaita faktūra 10 000 USD sumai.</span><span class="sxs-lookup"><span data-stu-id="e165d-116">The vendor sends you the 100 computers that have software installed, together with an invoice for 10,000.00 USD.</span></span> <span data-ttu-id="e165d-117">PWP sąlygos nustatytos jūsų projektui, todėl gavęs kompiuterius jūs tiekėjui nesumokate.</span><span class="sxs-lookup"><span data-stu-id="e165d-117">Because PWP terms are set up for your project, you don't pay the vendor upon receipt of the computers.</span></span> <span data-ttu-id="e165d-118">Vietoj to, jūs išsiunčiate kompiuterius klientui kartu su sąskaita faktūra 15 000 sumai.</span><span class="sxs-lookup"><span data-stu-id="e165d-118">Instead, you send the computers to the customer, together with an invoice for 15,000.00.</span></span> <span data-ttu-id="e165d-119">Klientas kompiuterius patikrina ir patvirtina visą projekto sąskaitos faktūros sumą.</span><span class="sxs-lookup"><span data-stu-id="e165d-119">The customer inspects the computers and approves the full amount of the project invoice.</span></span>

<span data-ttu-id="e165d-120">Iš kliento gavę visą mokėjimą, tiekėjui sumokate tiekėjui 10 000 – visą tiekėjo SF sumą.</span><span class="sxs-lookup"><span data-stu-id="e165d-120">After you receive the full payment from the customer, you pay the vendor 10,000.00, the full amount of the vendor invoice.</span></span>

## <a name="set-up-pwp-terms-for-a-project"></a><span data-ttu-id="e165d-121">Projekto PWP sąlygų nustatymas</span><span class="sxs-lookup"><span data-stu-id="e165d-121">Set up PWP terms for a project</span></span>

<span data-ttu-id="e165d-122">Kai nustatote projekto PWP sąlygas, turite nurodyti (procentais) minimalią sumą, kurią klientas turi jums sumokėti už projektą, prieš jums sumokant tiekėjui.</span><span class="sxs-lookup"><span data-stu-id="e165d-122">When you set up PWP terms for a project, you must specify, as a percentage, the minimum amount that a customer must pay you for the project before you will pay the vendor.</span></span> <span data-ttu-id="e165d-123">Ribinė suma automatiškai apskaičiuojama projekto kliento SF.</span><span class="sxs-lookup"><span data-stu-id="e165d-123">The threshold amount is automatically calculated on the customer invoices for the project.</span></span> <span data-ttu-id="e165d-124">Norėdami nustatyti PWP sąlygų ribinės vertės procentinį dydį, atlikite toliau nurodytus veiksmus.</span><span class="sxs-lookup"><span data-stu-id="e165d-124">Follow these steps to set up the threshold percentage for PWP terms.</span></span>

3. <span data-ttu-id="e165d-128">Lauke **Sąskaitos kodas** pasirinkite vieną iš toliau nurodytų parinkčių.</span><span class="sxs-lookup"><span data-stu-id="e165d-128">In the **Account code** field, select one of the following options:</span></span>

    - <span data-ttu-id="e165d-129">**Lentelė** – PWP sąlygos taikomos vienam tiekėjui.</span><span class="sxs-lookup"><span data-stu-id="e165d-129">**Table** – The PWP terms apply to a single vendor.</span></span>

5. <span data-ttu-id="e165d-134">Jei tiekėjui projekte nustatytos tiekėjo užlaikymo sąlygos, lauke **Tiekėjo užlaikymo sąlygos** pasirinkite užlaikymo sąlygų taisyklės ID.</span><span class="sxs-lookup"><span data-stu-id="e165d-134">If vendor retention terms are set up for the vendor in the project, in the **Vendor retention terms** field, select the rule ID for the retention terms.</span></span>
