## <a name="description-of-the-flushing-principles"></a><span data-ttu-id="9f380-125">Sunaudojimo principų aprašymas</span><span class="sxs-lookup"><span data-stu-id="9f380-125">Description of the flushing principles</span></span>

### <a name="manual"></a><span data-ttu-id="9f380-126">Neautomatinis</span><span class="sxs-lookup"><span data-stu-id="9f380-126">Manual</span></span>
<span data-ttu-id="9f380-127">Sunaudojimo principas Neautomatinis nurodo, kad medžiagų suvartojimo registravimas yra neautomatinė operacija.</span><span class="sxs-lookup"><span data-stu-id="9f380-127">The Manual flushing principle indicates that the registration of material consumption is a manual operation.</span></span> <span data-ttu-id="9f380-128">Šis principas yra aktualus, jei, pavyzdžiui, norite, kad būtų galima sekti laiką, o vartojamų paketų numerių arba serijos numerių kiekį reikia registruoti sekimo tikslais.</span><span class="sxs-lookup"><span data-stu-id="9f380-128">This principle is relevant if, for example, you want to be able to track time, and the quantity of consumed batch numbers or serial numbers must be accounted for, for tracking purposes.</span></span> <span data-ttu-id="9f380-129">Neautomatinis suvartojimas registruojamas gamybos išrinkimo dokumentų žurnale.</span><span class="sxs-lookup"><span data-stu-id="9f380-129">Manual consumption is registered in a production picking list journal.</span></span> <span data-ttu-id="9f380-130">Jei įjungta prekių patobulintų sandėlio procesų funkcija, galima taikyti rankinį srautą.</span><span class="sxs-lookup"><span data-stu-id="9f380-130">For items that are enabled for advanced warehouse processes, a hand-held flow can be applied.</span></span>

### <a name="start"></a><span data-ttu-id="9f380-131">Pradžios</span><span class="sxs-lookup"><span data-stu-id="9f380-131">Start</span></span>
<span data-ttu-id="9f380-132">Sunaudojimo principas Pradžia nurodo, kad medžiagos bus automatiškai suvartojamos pradėjus gamybos užsakymą.</span><span class="sxs-lookup"><span data-stu-id="9f380-132">The Start flushing principle indicates that material will be automatically consumed when the production order is started.</span></span> <span data-ttu-id="9f380-133">Suvartojamų medžiagų kiekis yra proporcingas pradžios kiekiui.</span><span class="sxs-lookup"><span data-stu-id="9f380-133">The amount of material that is consumed is proportional to the quantity that is started.</span></span> <span data-ttu-id="9f380-134">Jei sunaudojimo principas Pradžia yra naudojamas kartu su gamybos vykdymo sistema, jį taip pat galima naudoti norint sunaudoti medžiagas, kai pradedama operacija arba vykdymo užduotis.</span><span class="sxs-lookup"><span data-stu-id="9f380-134">When the Start flushing principle is used together with the manufacturing execution system, it can also be used to flush materials when an operation or a process job is started.</span></span> <span data-ttu-id="9f380-135">Šis principas yra aktualus, jei, pavyzdžiui, suvartojimo nuokrypis yra žemas, medžiagos yra žemos vertės medžiagos, sekimo reikalavimų nėra arba operacijų apdorojimo laikas yra trumpas.</span><span class="sxs-lookup"><span data-stu-id="9f380-135">This principle is relevant if, for example, the variance in the consumption is low, the materials are low-value materials, there are no tracking requirements, or there is a short run time on operations.</span></span> 

### <a name="finish"></a><span data-ttu-id="9f380-136">Baigti</span><span class="sxs-lookup"><span data-stu-id="9f380-136">Finish</span></span>
<span data-ttu-id="9f380-137">Sunaudojimo principas Pabaiga nurodo, kad medžiagos bus automatiškai suvartojamos, kai gamybos užsakymas paskelbiamas baigtu arba kai operacija, nustatyta vartoti medžiagas, užregistruojama kaip baigta.</span><span class="sxs-lookup"><span data-stu-id="9f380-137">The Finish flushing principle indicates that material will be automatically consumed when the production order is reported as finished, or when an operation that is set up to consume the materials is registered as completed.</span></span> <span data-ttu-id="9f380-138">Suvartojamų medžiagų kiekis yra proporcingas kiekiui, kuris paskelbtas baigtu.</span><span class="sxs-lookup"><span data-stu-id="9f380-138">The amount of material that is consumed is proportional to the quantity that is reported as finished.</span></span> <span data-ttu-id="9f380-139">Jei sunaudojimo principas Pabaiga yra naudojamas kartu su gamybos vykdymo sistema, jį taip pat galima naudoti norint sunaudoti medžiagas, kai baigiama operacija arba vykdymo užduotis.</span><span class="sxs-lookup"><span data-stu-id="9f380-139">When the Finish flushing principle is used together with the manufacturing execution system, it can also be used to flush materials when an operation or a process job is completed.</span></span> <span data-ttu-id="9f380-140">Šis principas yra aktualus tokiomis pačiomis sąlygomis kaip principas Pradžia.</span><span class="sxs-lookup"><span data-stu-id="9f380-140">This principle is relevant in the same situations as the Start principle.</span></span> <span data-ttu-id="9f380-141">Tačiau principas Pabaiga skirtas operacijoms, kurių apdorojimo laikas yra ilgesnis, kai medžiagos neturėtų būti nustatytos į parinktį NG, kol operacija nebaigta.</span><span class="sxs-lookup"><span data-stu-id="9f380-141">However, the Finish principle is for operations that have a longer run time, where materials should not be set to WIP before the operation is completed.</span></span> 

### <a name="available-at-location"></a><span data-ttu-id="9f380-142">Yra vietoje</span><span class="sxs-lookup"><span data-stu-id="9f380-142">Available at location</span></span>
<span data-ttu-id="9f380-143">Sunaudojimo principas Yra vietoje nurodo, kad medžiagos bus automatiškai suvartojamos, kai jos užregistruojamos kaip paimtos gamybai vykdyti.</span><span class="sxs-lookup"><span data-stu-id="9f380-143">The Available at location flushing principle indicates that the material will be automatically consumed when it's registered as picked for production.</span></span> <span data-ttu-id="9f380-144">Medžiagos užregistruojamos kaip paimtos iš vietos, kai žaliavų paėmimo užduotis baigta arba kai medžiagų yra gamybos įvesties vietoje ir medžiagų eilutė išleidžiama į sandėlį.</span><span class="sxs-lookup"><span data-stu-id="9f380-144">The material is registered as picked from location when work for the raw material picking is completed, or when material is available on the production input location and the material line is released to the warehouse.</span></span> <span data-ttu-id="9f380-145">Proceso metu sugeneruojamas išrinkimo dokumentas užregistruojamas paketinėje užduotyje.</span><span class="sxs-lookup"><span data-stu-id="9f380-145">The picking list that is generated during the process is posted in a batch job.</span></span> <span data-ttu-id="9f380-146">Šis principas yra aktualus, jei, pavyzdžiui, vienam gamybos užsakymui priskirta daug išrinkimo veiklų.</span><span class="sxs-lookup"><span data-stu-id="9f380-146">This principle is relevant if, for example, you have many picking activities against one production order.</span></span> <span data-ttu-id="9f380-147">Tokiu atveju išrinkimo dokumento neautomatiškai naujinti nereikia ir galima matyti dabartinį NG balanso rodinį.</span><span class="sxs-lookup"><span data-stu-id="9f380-147">In this case, you don't have to update the picking list manually, and you can get a current view of the WIP balance.</span></span>
