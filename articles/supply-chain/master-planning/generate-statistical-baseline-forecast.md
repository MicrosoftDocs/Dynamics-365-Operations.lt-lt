[!include [banner](../includes/banner.md)]

<span data-ttu-id="98c54-104">Šiame straipsnyje pateikiama informacija apie parametrus ir filtrus, kurie naudojami skaičiuojant poreikio prognozę.</span><span class="sxs-lookup"><span data-stu-id="98c54-104">This article provides information about the parameters and filters that are used in the calculation of demand forecasting.</span></span> 

<span data-ttu-id="98c54-105">Kuriant pagrindinę prognozę, pirmiausia reikia nurodyti skaičiuojant naudojamus parametrus ir filtrus.</span><span class="sxs-lookup"><span data-stu-id="98c54-105">When you create a baseline forecast, you must first specify the parameters and filters that are used in the calculation.</span></span> <span data-ttu-id="98c54-106">Pavyzdžiui, galite sukurti pagrindinę prognozę, kurioje poreikis vertinamas remiantis praėjusių metų operacijų duomenimis konkrečioje įmonėje, ateinančiam mėnesiui ir pasirinktai prekių grupei.</span><span class="sxs-lookup"><span data-stu-id="98c54-106">For example, you can create a baseline forecast that estimates demand based on transaction data from the past year for a specific company, for the coming month, and for a selected group of items.</span></span> 

<span data-ttu-id="98c54-107">Norėdami generuoti poreikio prognozę, pasirinkite **Bendrasis planavimas &gt; Prognozė &gt; Poreikio prognozė &gt; Generuoti pagrindinę statistinę prognozę**.</span><span class="sxs-lookup"><span data-stu-id="98c54-107">To generate a demand forecast, go to **Master planning &gt; Forecasting &gt; Demand forecasting &gt; Generate statistical baseline forecast**.</span></span> 

<span data-ttu-id="98c54-108">Prognozės rinkinį galima pasirinkti prognozės generavimo metu.</span><span class="sxs-lookup"><span data-stu-id="98c54-108">The forecast bucket can be selected at forecast generation time.</span></span> <span data-ttu-id="98c54-109">Galima naudoti šias reikšmes: Diena, Savaitė ir Mėnesis.</span><span class="sxs-lookup"><span data-stu-id="98c54-109">The available values are: Day, Week, and Month.</span></span> 

<span data-ttu-id="98c54-110">Lauke **Prognozės laikotarpis** nustatomas laikotarpių, kuriems reikia skaičiuoti prognozę, skaičių.</span><span class="sxs-lookup"><span data-stu-id="98c54-110">The number of buckets to generate a forecast for is set in the **Forecast horizon** field.</span></span> 

<span data-ttu-id="98c54-111">Kai prognozės strategija nustatyta kaip **Kopijuoti praeities poreikį**, praeities laikotarpio pabaigos nepaisoma.</span><span class="sxs-lookup"><span data-stu-id="98c54-111">When the forecast strategy is set to **Copy over historical demand**, the end of the historical horizon is ignored.</span></span> <span data-ttu-id="98c54-112">Rinkinių skaičių, nurodytą lauke **Prognozės laikotarpis**, sistema nukopijuoja į poreikio prognozę, pradedant nuo dalyje **Praeities laikotarpis** esančiame lauke **Pradžios data** nurodytos dienos.</span><span class="sxs-lookup"><span data-stu-id="98c54-112">The system copies the number of buckets specified in the **Forecast horizon** field to the forecast demand, starting from the date set in the **From date** field under **Historical horizon**.</span></span> <span data-ttu-id="98c54-113">Kopijuodami praeities poreikį nuo tam tikros datos į priekį, gamybos planuotojai gali kito ketvirčio planą sukurti dviem būdais:</span><span class="sxs-lookup"><span data-stu-id="98c54-113">By copying historical demand from a certain date forward, production planners can make the plan for the next quarter in two ways:</span></span>

-   <span data-ttu-id="98c54-114">kopijuodami poreikį iš to paties praėjusių metų ketvirčio;</span><span class="sxs-lookup"><span data-stu-id="98c54-114">By copying the demand from the same quarter last year.</span></span>
-   <span data-ttu-id="98c54-115">kopijuodami poreikį iš praėjusio ketvirčio.</span><span class="sxs-lookup"><span data-stu-id="98c54-115">By copying the demand from the previous quarter.</span></span>

<span data-ttu-id="98c54-116">Siekiant išvengti painiavos gamybos planuose, tam tikrą prognozės rinkinių skaičių galima užblokuoti.</span><span class="sxs-lookup"><span data-stu-id="98c54-116">To prevent confusion in the production plans, a certain number of forecast buckets can be frozen.</span></span> <span data-ttu-id="98c54-117">Šis skaičius nustatomas lauke **Blokavimo laiko ribos**.</span><span class="sxs-lookup"><span data-stu-id="98c54-117">This number is set in the **Freeze time fence** field.</span></span> <span data-ttu-id="98c54-118">Puslapyje **Pakoreguota poreikio prognozė** rodomi blokuojamų rinkinių langeliai vizualiai nurodo, kad tų reikšmių keisti negalima.</span><span class="sxs-lookup"><span data-stu-id="98c54-118">On the **Adjusted demand forecast** page, the cells for the frozen buckets are disabled, to give a visual indication that those values should not be changed.</span></span> 

<span data-ttu-id="98c54-119">Pagrindinės poreikio prognozės pradžios data neturi būti dabartinė data arba ateities data.</span><span class="sxs-lookup"><span data-stu-id="98c54-119">The start date for the baseline demand forecast doesn’t have to be the current date or a date in the future.</span></span> <span data-ttu-id="98c54-120">Norėdami nustatyti kitą pradžios datą, naudokite lauką **Pagrindinės prognozės pradžios data – pradžios data**.</span><span class="sxs-lookup"><span data-stu-id="98c54-120">To set a different start date, use the **Baseline forecast start date - From date** field.</span></span> <span data-ttu-id="98c54-121">Pavyzdžiui, birželį vartotojai gali generuoti kitų metų prognozę.</span><span class="sxs-lookup"><span data-stu-id="98c54-121">For example, in June, users can generate a forecast for the next year.</span></span> <span data-ttu-id="98c54-122">Kadangi nenurodyti praeities poreikio pabaigos ir pagrindinės prognozės pradžios prognozės rinkiniai, prognozės gali būti netikslios.</span><span class="sxs-lookup"><span data-stu-id="98c54-122">Because the forecast buckets between the end of historical demand and the start of the baseline are missing, the predictions might not be accurate.</span></span> <span data-ttu-id="98c54-123">Jei naudojate „Microsoft Dynamics 365 for Finance and Operations“ poreikio prognozės tarnybą, trūkstamą informaciją galite įvesti keturiais būdais.</span><span class="sxs-lookup"><span data-stu-id="98c54-123">If you are using the Microsoft Dynamics 365 for Finance and Operations Demand forecasting service, there are four ways in which you can fill in the missing gaps.</span></span> <span data-ttu-id="98c54-124">Norimą metodą galite pasirinkti puslapyje **Poreikio prognozės parametrai** nustatydami parametrą MISSING\_VALUE\_SUBSTITUTION.</span><span class="sxs-lookup"><span data-stu-id="98c54-124">You can choose the method that you want by setting the MISSING\_VALUE\_SUBSTITUTION parameter on the **Demand forecasting parameters** page.</span></span> 

<span data-ttu-id="98c54-125">Lauko **Pagrindinės prognozės pradžios data** -  **Pradžios data** reikšmė turi būti nustatyta prognozės rinkinio pradžioje, pvz., Jungtinėse Amerikos Valstijose tai būtų sekmadienis, jei prognozės rinkinys yra savaitė.</span><span class="sxs-lookup"><span data-stu-id="98c54-125">The **Baseline forecast start date** - **From date** field has to be set to the beginning of a forecast bucket, for example, in the United States, a Sunday if the forecasting bucket is the week.</span></span> <span data-ttu-id="98c54-126">Sistema automatiškai koreguoja lauko **Pagrindinės prognozės pradžios data**  -  **Pradžios data** reikšmę, kad ji atitiktų prognozės rinkinio pradžią.</span><span class="sxs-lookup"><span data-stu-id="98c54-126">The system automatically adjusts the **Baseline forecast start date** - **From date** field to match the beginning of a forecast bucket.</span></span> 

<span data-ttu-id="98c54-127">Lauko **Pagrindinės prognozės pradžios data** -  **Pradžios data** reikšmę galima nustatyti kaip praeities datą.</span><span class="sxs-lookup"><span data-stu-id="98c54-127">The **Baseline forecast start date** - **From date** field can be set to a date in in the past.</span></span> <span data-ttu-id="98c54-128">Kitaip tariant, galima generuoti praeities poreikio prognozę.</span><span class="sxs-lookup"><span data-stu-id="98c54-128">In other words, it is possible to generate a demand forecast in the past.</span></span> <span data-ttu-id="98c54-129">Tai naudinga, nes vartotojai gali keisti prognozės tarnybos parametrus, kad anksčiau sugeneruota statistinė prognozė atitiktų faktinį praeities poreikį.</span><span class="sxs-lookup"><span data-stu-id="98c54-129">This is useful, because it lets users tweak the forecast service parameters so that the statistical forecast generated in the past matches the actual historical demand.</span></span> <span data-ttu-id="98c54-130">Tada vartotojai gali toliau naudoti šį parametrų nustatymą, norėdami generuoti ateities pagrindinę statistinę prognozę.</span><span class="sxs-lookup"><span data-stu-id="98c54-130">Users can then continue using these parameter settings to generate a statistical baseline forecast for the future.</span></span> 

<span data-ttu-id="98c54-131">Ankstesnių poreikio prognozavimo pakartojimų koregavimus, atliktus neautomatiniu būdu, galima automatiškai taikyti naujai pagrindinei prognozei, jei pažymėtas žymės be automatically applied to the new baseline forecast if the **Transfer manual adjustments to the demand forecas**t check box is selected.</span></span> <span data-ttu-id="98c54-132">Jei žymės langelis išvalytas, neautomatiniai koregavimai neįtraukiami į pagrindinę prognozę, bet nėra panaikinami.</span><span class="sxs-lookup"><span data-stu-id="98c54-132">If the check box is cleared, the manual adjustments are not added to the baseline forecast – but they are not deleted.</span></span> <span data-ttu-id="98c54-133">Neautomatinius prognozės koregavimus galima panaikinti tik prognozės made to a forecast can be deleted only at forecast import time, by clearing the **Save the manual adjustments made to the baseline demand forecast** check box.</span></span> <span data-ttu-id="98c54-134">Neautomatiniai koregavimai įrašomi autorizavimo metu.</span><span class="sxs-lookup"><span data-stu-id="98c54-134">Manual adjustments are saved at authorization time.</span></span> <span data-ttu-id="98c54-135">Todėl jei vartotojas neautomatiškai koreguoja prognozę, tačiau neįgalioja jos „Finance and Operations“, pakeitimai bus prarasti.</span><span class="sxs-lookup"><span data-stu-id="98c54-135">Therefore, if a user makes manual adjustments to the forecast, but doesn’t authorize the forecast back to Finance and Operations, the changes are lost.</span></span> <span data-ttu-id="98c54-136">Daugiau informacijos apie neautomatinius koregavimus ir jų paskirtį žr. puslapyje [Pakoreguotos prognozės <span data-ttu-id="98c54-137">Poreikio prognozės generavimo procesas gali turėti pavadinimą ir komentarų, kad vartotojai galėtų lengviau identifikuoti sugeneruotą prognozę.</span><span class="sxs-lookup"><span data-stu-id="98c54-137">A demand forecast generation can have a name and comments to help users identify the forecast that has been generated.</span></span> <span data-ttu-id="98c54-138">Šios reikšmės yra matomos prognozės generavimo retrospektyvoje puslapyje **Pagrindinės statistinės prognozės generavimo retrospektyva**.</span><span class="sxs-lookup"><span data-stu-id="98c54-138">These values are visible in forecast generation history on the **Statistical baseline forecast generation history** page.</span></span> 

<span data-ttu-id="98c54-139">Vidinės įmonės planavimo grupės, prekių paskirstymo raktų ir kitus filtrus galima taikyti prognozės generavimo metu.</span><span class="sxs-lookup"><span data-stu-id="98c54-139">The intercompany planning group, item allocation keys, and other filters can be applied at forecast generation time.</span></span> <span data-ttu-id="98c54-140">Juos galima naudoti našumui pagerinti arba duomenims į įvykdomas dalis padalinti.</span><span class="sxs-lookup"><span data-stu-id="98c54-140">These can be used to improve performance or to split the data into manageable chunks.</span></span> <span data-ttu-id="98c54-141">Tačiau atkreipkite dėmesį, kad poreikio prognozė nėra generuojama su vidinės įmonės planavimo grupe nesusieto prekių paskirstymo rakto nariams, net jei užklausoje prekių paskirstymo raktas yra pasirinktas.</span><span class="sxs-lookup"><span data-stu-id="98c54-141">However, note that a demand forecast is not generated for the members of any item allocation key that is not associated with an intercompany planning group, even if the item allocation key is selected in the query.</span></span>
