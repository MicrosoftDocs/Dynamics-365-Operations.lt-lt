<span data-ttu-id="48553-104">Standartinių savikainos atnaujinimų duomenis galima valdyti dviem būdais: vienos versijos būdu ir dviejų versijų būdu.</span><span class="sxs-lookup"><span data-stu-id="48553-104">Updates to standard cost data can be managed by using two different approaches -  the one-version approach and the two-version approach.</span></span> 

<span data-ttu-id="48553-105">Vienos versijos būdui naudojama viena savikainos nustatymo versija, kurioje įtraukiama visų išlaidų apskaita.</span><span class="sxs-lookup"><span data-stu-id="48553-105">The one-version approach uses a single costing version that contains all cost records.</span></span> <span data-ttu-id="48553-106">Šie įrašai apima pradinę savikainą ir visus savikainos atnaujinimus.</span><span class="sxs-lookup"><span data-stu-id="48553-106">These records include the original costs and all cost updates.</span></span>
<span data-ttu-id="48553-107">Dviejų versijų būdui naudojama viena versija, kurioje yra pradinės savikainos įrašai, ir antroji versija, kurioje yra visų savikainos atnaujinimų įrašai.</span><span class="sxs-lookup"><span data-stu-id="48553-107">The two-version approach uses one version that contains records of the original costs and a second version that contains records of all cost originalios įkainojimo versijos.</span><span class="sxs-lookup"><span data-stu-id="48553-108">A primary advantage of the two-version approach is the clear delineation and tracking of cost updates in a aplinkoje.</span><span class="sxs-lookup"><span data-stu-id="48553-110">**Example** The following example illustrates how the one-version and two-version approaches can be used for updating standard costs in a manufacturing environment.</span></span> <span data-ttu-id="48553-111">Pavyzdžiui, atnaujinimai, atspindintys naujus elementus arba klaidų pataisymus.</span><span class="sxs-lookup"><span data-stu-id="48553-111">For example, updates that reflect new items or error corrections.</span></span> <span data-ttu-id="48553-112">Tarkime, kad vienos versijos įkainojimas apima standartinę data-ttu-id="48553-113">Šios versijos identifikatorius yra 2016-STD.</span><span class="sxs-lookup"><span data-stu-id="48553-113">The identifier for this version is 2016-STD.</span></span> <span data-ttu-id="48553-114">Versija 2016-STD apima esamas aktyvias visų elementų savikainas.</span><span class="sxs-lookup"><span data-stu-id="48553-114">Version 2016-STD contains the current active costs -   <span data-ttu-id="48553-117">**Savikainos duomenų naujinimo vienos versijos būdas** − Vienos versijos būdas reiškia, kad originalioje įkainojimo versijoje 2016-STD bus visi savikainos įrašai.</span><span class="sxs-lookup"><span data-stu-id="48553-117">**One-version approach to cost data updates** − In the one-version approach, the original costing version 2016-STD contains all cost records.</span></span> <span data-ttu-id="48553-118">Savikainos atnaujinimai yra įrašyti 2016-STD ir nustatyti „Laukiama“ būsenai.</span><span class="sxs-lookup"><span data-stu-id="48553-118">Cost updates are recorded in 2016-STD and are set to a status of ”Pending.”</span></span> <span data-ttu-id="48553-119">Laukiamą savikainą galima rankiniu būdu įvesti naujoms įsigytoms prekėms arba apskaičiuoti gatavam gaminiui, siekiant atspindėti pataisymus. <span data-ttu-id="48553-129">Šiame pavyzdyje naudojama viena papildoma įkainojimo versija naujinimų valdymui 2016 m.</span><span class="sxs-lookup"><span data-stu-id="48553-129">This example used one additional costing version for managing updates throughout the year 2016.</span></span> <span data-ttu-id="48553-130">Gali būti naudojama daugiau nei viena papildoma įkainojimo versija, pavyzdžiui, atskira versija kiekvienam naujinimo paketui.</span><span class="sxs-lookup"><span data-stu-id="48553-130">More than one additional costing version can be used, such as a separate version for each batch of updates.</span></span> <span data-ttu-id="48553-131">Kai naudojamas daugiau nei vienas papildomas sąnaudų skaičiavimas, atsarginius duomenis reikia išreikšti kaip aktyvią savikainą, nes aktyvi savikaina paskirstoma kelioms savikainos versijoms.</span><span class="sxs-lookup"><span data-stu-id="48553-131">When more than one additional costing is used, the fallback must be expressed as the active costs, because the active costs are spread over multiple costing versions.</span></span>
