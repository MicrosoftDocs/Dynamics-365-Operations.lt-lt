# <a name="manage-standard-cost-updates"></a><span data-ttu-id="5e86a-103">Standartinių savikainos atnaujinimų valdymas</span><span class="sxs-lookup"><span data-stu-id="5e86a-103">Manage standard cost updates</span></span>

[!include [banner](../includes/banner.md)]

<span data-ttu-id="5e86a-104">Standartinių savikainos atnaujinimų duomenis galima valdyti dviem būdais: vienos versijos būdu arba dviejų versijų būdu.</span><span class="sxs-lookup"><span data-stu-id="5e86a-104">Updates to standard cost data can be managed by using two different approaches - the one-version approach or the two-version approach.</span></span> 

<span data-ttu-id="5e86a-105">Vienos versijos būdui naudojama viena savikainos nustatymo versija, kurioje įtraukiama visų išlaidų apskaita.</span><span class="sxs-lookup"><span data-stu-id="5e86a-105">The one-version approach uses a single costing version that contains all cost records.</span></span> <span data-ttu-id="5e86a-106">Šie įrašai apima pradinę savikainą ir visus savikainos atnaujinimus.</span><span class="sxs-lookup"><span data-stu-id="5e86a-106">These records include the original costs and all cost updates.</span></span>

<span data-ttu-id="5e86a-107">Dviejų versijų būdui naudojama viena versija, kurioje yra pradinės savikainos įrašai, ir antroji versija, kurioje yra visų savikainos atnaujinimų įrašai.</span><span class="sxs-lookup"><span data-stu-id="5e86a-107">The two-version approach uses one version that contains records of the original costs and a second version that contains records of all cost updates.</span></span> <span data-ttu-id="5e86a-108">Svarbiausias dviejų versijų būdo pranašumas yra aiškus savikainų naujinimų apibrėžimas ir sekimas atskirose įkainojimo versijose, neįtakojant originalios įkainojimo versijos.</span><span class="sxs-lookup"><span data-stu-id="5e86a-108">A primary advantage of the two-version approach is the clear delineation and tracking of cost updates in a separate costing version, without affecting the original costing version.</span></span> <span data-ttu-id="5e86a-109">Dviejų versijų principas gali būti naudojamas identifikuoti kelis papildančiuosius naujinimus, kur kiekvienas papildantysis naujinimas turi atskirą įkainojimo versiją, susidedančią iš papildančiosios savikainos įrašų.</span><span class="sxs-lookup"><span data-stu-id="5e86a-109">The two-version approach can be used to identify multiple incremental updates, where each incremental update has a separate costing version that contains the incremental cost records.</span></span> 

<span data-ttu-id="5e86a-110">**Pavyzdys**</span><span class="sxs-lookup"><span data-stu-id="5e86a-110">**Example**</span></span> 

<span data-ttu-id="5e86a-111">Šiame pavyzdyje parodyta, kaip galima naudoti vienos ir dviejų versijų būdus standartinėms savikainoms gamybinėje aplinkoje naujinti.</span><span class="sxs-lookup"><span data-stu-id="5e86a-111">The following example illustrates how the one-version and two-version approaches can be used for updating standard costs in a manufacturing environment.</span></span> <span data-ttu-id="5e86a-112">Pavyzdžiui, atnaujinimai, atspindintys naujus elementus arba klaidų pataisymus.</span><span class="sxs-lookup"><span data-stu-id="5e86a-112">For example, updates that reflect new items or error corrections.</span></span> <span data-ttu-id="5e86a-113">Tarkime, kad vienos versijos įkainojimas apima standartinę savikainą einamiesiems metams.</span><span class="sxs-lookup"><span data-stu-id="5e86a-113">Assume that a single costing version represents the standard costs for the current year.</span></span> <span data-ttu-id="5e86a-114">Šios versijos identifikatorius yra 2016-STD.</span><span class="sxs-lookup"><span data-stu-id="5e86a-114">The identifier for this version is 2016-STD.</span></span> <span data-ttu-id="5e86a-115">Versija 2016-STD apima esamas aktyvias visų elementų savikainas.</span><span class="sxs-lookup"><span data-stu-id="5e86a-115">Version 2016-STD contains the current active costs for all items.</span></span> <span data-ttu-id="5e86a-116">Be to, ją sudaro visų su paskirstymu susijusių savikainų kategorijos ir pridėtinių išlaidų skaičiavimo formulės kurios yra žinomos 2016 m. pradžioje.</span><span class="sxs-lookup"><span data-stu-id="5e86a-116">Additionally, it contains all routing-related cost categories and overhead calculation formulas that were known at the start of the year 2016.</span></span> <span data-ttu-id="5e86a-117">2016-STD yra pradinė savikainos nustatymo versija.</span><span class="sxs-lookup"><span data-stu-id="5e86a-117">2016-STD is the original costing version.</span></span>

-   <span data-ttu-id="5e86a-118">**Savikainos duomenų naujinimo vienos versijos būdas** − Vienos versijos būdas reiškia, kad originalioje įkainojimo versijoje 2016-STD bus visi savikainos įrašai.</span><span class="sxs-lookup"><span data-stu-id="5e86a-118">**One-version approach to cost data updates** − In the one-version approach, the original costing version 2016-STD contains all cost records.</span></span> <span data-ttu-id="5e86a-119">Savikainos atnaujinimai yra įrašyti 2016-STD ir nustatyti „Laukiama“ būsenai.</span><span class="sxs-lookup"><span data-stu-id="5e86a-119">Cost updates are recorded in 2016-STD and are set to a status of ”Pending.”</span></span> <span data-ttu-id="5e86a-120">Naujai įsigytų prekių laukiamą savikainą galima įvesti neautomatiškai arba apskaičiuoti pagamintos prekės savikainą, siekiant atspindėti pataisymus.</span><span class="sxs-lookup"><span data-stu-id="5e86a-120">The pending costs can be manually entered for new purchased items, or they can be calculated for a manufactured item to reflect corrections.</span></span> <span data-ttu-id="5e86a-121">Naudojant vienos versijos būdą, KS skaičiavimams atlikti nereikalingas atsarginis duomenų šaltinis, nes visos aktyvios savikainos yra įkainojimo versijoje.</span><span class="sxs-lookup"><span data-stu-id="5e86a-121">When the one-version approach is used, the BOM calculations do not require a fallback data source because all active costs are contained in the costing version.</span></span> <span data-ttu-id="5e86a-122">Aktyvavus laukiančiąsias savikainas, originali įkainojimo versija 2016-STD dar kartą įtrauks visas esamas aktyvias savikainas.</span><span class="sxs-lookup"><span data-stu-id="5e86a-122">After the pending costs become active, the original costing version 2016-STD will again contain all the current active costs.</span></span>
-   <span data-ttu-id="5e86a-123">**Savikainos duomenų naujinimo dviejų versijų būdas** − dviejų versijų būdui reikia papildomų sąnaudų versijos kurioje yra tik sąnaudų atnaujinimai.</span><span class="sxs-lookup"><span data-stu-id="5e86a-123">**Two-version approach to cost data updates** − The two-version approach requires an additional costing version that contains only the cost updates.</span></span> <span data-ttu-id="5e86a-124">Šios versijos identifikatorius yra 2016-STD-CHANGES.</span><span class="sxs-lookup"><span data-stu-id="5e86a-124">The identifier for this version is 2016-STD-CHANGES.</span></span> <span data-ttu-id="5e86a-125">Savikainos atnaujinimai yra įrašyti 2016-STD-CHANGES ir jiems nustatyta būsena „Laukiama“.</span><span class="sxs-lookup"><span data-stu-id="5e86a-125">Cost updates are recorded in 2016-STD-CHANGES and are set to a status of “Pending.”</span></span> <span data-ttu-id="5e86a-126">Naudojant dviejų versijų būdą, pagamintų prekių data.</span></span> <span data-ttu-id="5e86a-128">Atsarginis skaičiavimas gali būti išreikštas kaip aktyvi savikaina ar kaip įkainojimo versija 2016-STD, nes abi jos nustato savikainos duomenų šaltinį, kurio 2016-STD-CHANGES nėra.</span><span class="sxs-lookup"><span data-stu-id="5e86a-128">The fallback can be expressed as the active costs or as the costing version 2016-STD, because both costs become active, the costing version 2016-STD-CHANGES will contain the current active costs that reflect the updates, whereas the original costing version 2016-STD will be untouched.</span></span> <span data-ttu-id="5e86a-130">Kai naudojamas dviejų versijų būdas, originalios įkainojimo versijos blokuojančiosios strategijos nustatomos taip, kad uždraustų naujinimus.</span><span
